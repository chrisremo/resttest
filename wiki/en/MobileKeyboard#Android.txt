<div id='wikitext'>
<h1>Mobile Keyboard</h1>
<p>In most cases, Unity will handle keyboard input automatically for GUI elements but it is also easy to show the keyboard on demand from a script.
</p>
<div class='vspace'></div><a class='iosRef' id='iosRef' href="javascript:ShowHideDiv('iosRef','specific-ios', 'iOS');"><img src="./uploads/Main/PlatformArrowDown.png"><h1>iOS</h1></a>
<p><a name='iOS' id='iOS'></a>
</p><div class='specific-ios' style='display: block;' id='specific-ios' > 
<h2>Using the Keyboard</h2>
<h3>GUI Elements</h3>
<p>The keyboard will appear automatically when a user taps on editable GUI elements. Currently, <a class='urllink' href='../ScriptReference/GUI.TextField.html' rel='nofollow'>GUI.TextField</a>, <a class='urllink' href='../ScriptReference/GUI.TextArea.html' rel='nofollow'>GUI.TextArea</a> and <a class='urllink' href='../ScriptReference/GUI.PasswordField.html' rel='nofollow'>GUI.PasswordField</a> will display the keyboard; see the  <a class='urllink' href='../ScriptReference/GUI.html' rel='nofollow'>GUI class</a> documentation for further details.
</p>
<div class='vspace'></div><h3>Manual Keyboard Handling</h3>
<p>Use the <span class='doc-prop'>iPhoneKeyboard.Open</span> function to open the keyboard. Please see the <a class='urllink' href='../ScriptReference/iPhoneKeyboard.html' rel='nofollow'>iPhoneKeyboard</a> scripting reference for the parameters that this function takes.
</p>
<div class='vspace'></div><h2>Keyboard Type Summary</h2>
<p>The Keyboard supports the following types:
</p><table width='100%'><tr ><th  align='left'>Property:</th><th  align='left'>Function:</th></tr>
<tr ><td  align='left'><span class='doc-prop'>iPhoneKeyboardType.Default</span></td><td >Letters. Can be switched to keyboard with numbers and punctuation.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>iPhoneKeyboardType.ASCIICapable</span></td><td >Letters. Can be switched to keyboard with numbers and punctuation.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>iPhoneKeyboardType.NumbersAndPunctuation</span></td><td >Numbers and punctuation. Can be switched to keyboard with letters.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>iPhoneKeyboardType.URL</span></td><td >Letters with slash and .com buttons. Can be switched to keyboard with numbers and punctuation.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>iPhoneKeyboardType.NumberPad</span></td><td >Only numbers from 0 to 9.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>iPhoneKeyboardType.PhonePad</span></td><td >Keyboard used to enter phone numbers.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>iPhoneKeyboardType.NamePhonePad</span></td><td >Letters. Can be switched to phone keyboard.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>iPhoneKeyboardType.EmailAddress</span></td><td >Letters with @ sign. Can be switched to keyboard with numbers and punctuation.</td></tr>
</table>
<div class='vspace'></div><h2>Text Preview</h2>
<p>By default, an edit box will be created and placed on top of the keyboard after it appears. This works as preview of the text that user is typing, so the text is always visible for the user. However, you can disable text preview by setting <span class='doc-prop'>iPhoneKeyboard.hideInput</span> to true. Note that this works only for certain keyboard types and input modes. For example, it will not work for phone keypads and multi-line text input. In such cases, the edit box will always appear. <span class='doc-prop'>iPhoneKeyboard.hideInput</span> is a global variable and will affect all keyboards.
</p>
<div class='vspace'></div><h2>Keyboard Orientation</h2>
<p>By default, the keyboard automatically follows the device orientation. To disable or enable rotation to a certain orientation, use the following properties available in <a class='urllink' href='../ScriptReference/iPhoneKeyboard.html' rel='nofollow'>iPhoneKeyboard</a>:
</p><table width='100%'><tr ><th  align='left'>Property:</th><th  align='left'>Function:</th></tr>
<tr ><td  align='left'><span class='doc-prop'>autorotateToPortrait</span></td><td >Enable or disable autorotation to portrait orientation (button at the bottom).</td></tr>
<tr ><td  align='left'><span class='doc-prop'>autorotateToPortraitUpsideDown</span></td><td >Enable or disable autorotation to portrait orientation (button at top).</td></tr>
<tr ><td  align='left'><span class='doc-prop'>autorotateToLandscapeLeft</span></td><td >Enable or disable autorotation to landscape left orientation (button on the right).</td></tr>
<tr ><td  align='left'><span class='doc-prop'>autorotateToLandscapeRight</span></td><td >Enable or disable autorotation to landscape right orientation (button on the left).</td></tr>
</table>
<div class='vspace'></div><h2>Visibility and Keyboard Size</h2>
<p>There are three keyboard properties in <a class='urllink' href='../ScriptReference/iPhoneKeyboard.html' rel='nofollow'>iPhoneKeyboard</a> that determine keyboard visibility status and size on the screen.
</p><table width='100%'><tr ><th  align='left'>Property:</th><th  align='left'>Function:</th></tr>
<tr ><td  align='left'><span class='doc-prop'>visible</span></td><td >Returns <span class='doc-prop'>true</span> if the keyboard is fully visible on the screen and can be used to enter characters.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>area</span></td><td >Returns the position and dimensions of the keyboard.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>active</span></td><td >Returns <span class='doc-prop'>true</span> if the keyboard is activated. This property is not static property. You must have a keyboard instance to use this property.</td></tr>
</table>
<p class='vspace'>Note that <span class='doc-prop'>iPhoneKeyboard.area</span> will return a rect with position and size set to 0 until the keyboard is fully visible on the screen. You should not query this value immediately after <span class='doc-prop'>iPhoneKeyboard.Open</span>. The sequence of keyboard events is as follows:
</p>
<div class='vspace'></div><ul><li><span class='doc-prop'>iPhoneKeyboard.Open</span> is called. <span class='doc-prop'>iPhoneKeyboard.active</span> returns true. <span class='doc-prop'>iPhoneKeyboard.visible</span> returns false. <span class='doc-prop'>iPhoneKeyboard.area</span> returns (0, 0, 0, 0).
</li><li>Keyboard slides out into the screen. All properties remain the same.
</li><li>Keyboard stops sliding. <span class='doc-prop'>iPhoneKeyboard.active</span> returns true. <span class='doc-prop'>iPhoneKeyboard.visible</span> returns true. <span class='doc-prop'>iPhoneKeyboard.area</span> returns real position and size of the keyboard.
</li></ul><div class='vspace'></div><h2>Secure Text Input</h2>
<p>It is possible to configure the keyboard to hide symbols when typing. This is useful when users are required to enter sensitive information (such as passwords). To manually open keyboard with secure text input enabled, use the following code:
</p><pre class='escaped'>
iPhoneKeyboard.Open("", iPhoneKeyboardType.Default, false, false, true);
</pre>
<div class='vspace'></div><div><img src='http://docwiki.unity3d.co.jp/uploads/Main/KeyboardSecure.png' alt='' title='' /></div>
<p><em>Hiding text while typing</em>
</p>
<div class='vspace'></div><h2>Alert keyboard</h2>
<p>To display the keyboard with a black semi-transparent background instead of the classic opaque, call <span class='doc-prop'>iPhoneKeyboard.Open</span> as follows:
</p><pre class='escaped'>
iPhoneKeyboard.Open("", iPhoneKeyboardType.Default, false, false, true, true);
</pre>
<div class='vspace'></div><div><img src='http://docwiki.unity3d.co.jp/uploads/Main/KeyboardClassic.png' alt='' title='' /></div>
<p><em>Classic keyboard</em>
</p>
<div class='vspace'></div><div><img src='http://docwiki.unity3d.co.jp/uploads/Main/KeyboardAlert.png' alt='' title='' /></div>
<p><em>Alert keyboard</em>
</p>
<div class='vspace'></div></div>
<div class='vspace'></div><a class='androidRef' id='androidRef' href="javascript:ShowHideDiv('androidRef','specific-android', 'Android');"><img src="./uploads/Main/PlatformArrowDown.png"><h1>Android</h1></a>
<p><a name='Android' id='Android'></a>
</p><div class='specific-android' style='display: block;' id='specific-android' > 
<p>Unity Android reuses the iOS API to display system keyboard. Even though Unity Android supports most of the functionality of its iPhone counterpart, there are two aspects which are not supported:
</p><ul><li><span class='doc-prop'>iPhoneKeyboard.hideInput</span>
</li><li><span class='doc-prop'>iPhoneKeyboard.area</span>
</li></ul><p class='vspace'>Please also note that the layout of a <span class='doc-prop'>iPhoneKeyboardType</span> can differ somewhat between devices.
</p></div>
