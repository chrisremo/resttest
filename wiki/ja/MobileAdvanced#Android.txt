<div id='wikitext'>
<h1>Advanced Unity Mobile Scripting</h1>
<p><a name='iOS' id='iOS'></a>
</p><a class='iosRef' id='iosRef' href="javascript:ShowHideDiv('iosRef','specific-ios', 'iOS');"><img src="./uploads/Main/PlatformArrowDown.png"><h1>iOS</h1></a>
<div class='specific-ios' style='display: block;' id='specific-ios' > 
<h1>Advanced iOS scripting</h1>
<h2>Determining Device Generation</h2>
<p>Different device generations support different functionality and have widely varying performance. You should query the device's generation and decide which functionality should be disabled to compensate for slower devices.
</p>
<p class='vspace'>You can find the device generation from the <a class='urllink' href='../ScriptReference/iPhone-generation.html' rel='nofollow'>iPhone.generation</a> property. The reported generation can be one of the following:
</p><ul><li><span class='doc-prop'>iPhone</span>
</li><li><span class='doc-prop'>iPhone3G</span>
</li><li><span class='doc-prop'>iPhone3GS</span>
</li><li><span class='doc-prop'>iPhone4</span>
</li><li><span class='doc-prop'>iPodTouch1Gen</span>
</li><li><span class='doc-prop'>iPodTouch2Gen</span>
</li><li><span class='doc-prop'>iPodTouch3Gen</span>
</li><li><span class='doc-prop'>iPodTouch4Gen</span>
</li><li><span class='doc-prop'>iPad1Gen</span>
</li></ul><p class='vspace'>You can find more information about different device generations, performance and supported functionality in our <a class='wikilink' href='http://docwiki.unity3d.co.jp/index.php?n=Main.iphone-Hardware'>iPhone Hardware Guide</a>.
</p>
<div class='vspace'></div><h2>Device Properties</h2>
<p>There are a number of device-specific properties that you can access:-
</p><table width='100%'><tr ><th  align='left'>Property:</th><th  align='left'>Function:</th></tr>
<tr ><td  align='left'><span class='doc-prop'><a class='urllink' href='../ScriptReference/SystemInfo-deviceUniqueIdentifier.html' rel='nofollow'>SystemInfo.deviceUniqueIdentifier</a></span></td><td >Unique device identifier.</td></tr>
<tr ><td  align='left'><span class='doc-prop'><a class='urllink' href='../ScriptReference/SystemInfo-deviceName.html' rel='nofollow'>SystemInfo.deviceName</a></span></td><td >User specified name for device.</td></tr>
<tr ><td  align='left'><span class='doc-prop'><a class='urllink' href='../ScriptReference/SystemInfo-deviceModel.html' rel='nofollow'>SystemInfo.deviceModel</a></span></td><td >Is it iPhone or iPod Touch?</td></tr>
<tr ><td  align='left'><span class='doc-prop'><a class='urllink' href='../ScriptReference/SystemInfo-operatingSystem.html' rel='nofollow'>SystemInfo.operatingSystem</a></span></td><td >Operating system name and version.</td></tr>
</table>
<div class='vspace'></div><h2>Anti-Piracy Check</h2>
<p>Pirates will often hack an application from the AppStore (by removing Apple DRM protection) and then redistribute it for free. Unity iOS comes with an anti-piracy check which allows you to determine if your application was altered <b>after</b> it was submitted to the AppStore.
</p>
<p class='vspace'>You can check if your application is genuine (not-hacked) with the <a class='urllink' href='../ScriptReference/Application-genuine.html' rel='nofollow'>Application.genuine</a> property. If this property returns <span class='doc-keyword'>false</span> then you might notify the user that he is using a hacked application or maybe disable access to some functions of your application.
</p>
<p class='vspace'><b>Note:</b> accessing the <a class='urllink' href='../ScriptReference/Application-genuine.html' rel='nofollow'>Application.genuine</a> property is a fairly expensive operation and so you shouldn't do it during frame updates or other time-critical code. 
</p>
<div class='vspace'></div><h2>Vibration Support</h2>
<p>You can trigger a vibration by calling <a class='urllink' href='../ScriptReference/Handheld.Vibrate.html' rel='nofollow'>Handheld.Vibrate</a>. Note that iPod Touch devices lack vibration hardware and will just ignore this call.
</p>
<div class='vspace'></div></div>
<p class='vspace'><a name='Android' id='Android'></a>
</p><a class='androidRef' id='androidRef' href="javascript:ShowHideDiv('androidRef','specific-android', 'Android');"><img src="./uploads/Main/PlatformArrowDown.png"><h1>Android</h1></a>
<div class='specific-android' style='display: block;' id='specific-android' > 
<h1>Advanced Android scripting</h1>
<h2>Determining Device Generation</h2>
<p>Different Android devices support different functionality and have widely varying performance. You should target specific devices or device families and decide which functionality should be disabled to compensate for slower devices. There are a number of device specific properties that you can access to which device is being used.
</p>
<p class='vspace'><b>Note:</b> Android Marketplace does some additional compatibility filtering, so you should not be concerned if an ARMv7-only app optimised for OGLES2 is offered to some old slow devices.
</p>
<div class='vspace'></div><h2>Device Properties</h2>
<table ><tr ><th  align='left'>Property:</th><th  align='left'>Function:</th></tr>
<tr ><td  align='left'><span class='doc-prop'><a class='urllink' href='../ScriptReference/SystemInfo-deviceUniqueIdentifier.html' rel='nofollow'>SystemInfo.deviceUniqueIdentifier</a></span></td><td >Unique device identifier.</td></tr>
<tr ><td  align='left'><span class='doc-prop'><a class='urllink' href='../ScriptReference/SystemInfo-deviceName.html' rel='nofollow'>SystemInfo.deviceName</a></span></td><td >User specified name for device.</td></tr>
<tr ><td  align='left'><span class='doc-prop'><a class='urllink' href='../ScriptReference/SystemInfo-deviceModel.html' rel='nofollow'>SystemInfo.deviceModel</a></span></td><td >Is it iPhone or iPod Touch?</td></tr>
<tr ><td  align='left'><span class='doc-prop'><a class='urllink' href='../ScriptReference/SystemInfo-operatingSystem.html' rel='nofollow'>SystemInfo.operatingSystem</a></span></td><td >Operating system name and version.</td></tr>
</table>
<div class='vspace'></div><h2>Anti-Piracy Check</h2>
<p>Pirates will often hack an application (by removing Apple DRM protection) and then redistribute it for free. Unity Android comes with an anti-piracy check which allows you to determine if your application was altered <b>after</b> it was submitted to the AppStore.
</p>
<p class='vspace'>You can check if your application is genuine (not-hacked) with the <a class='urllink' href='../ScriptReference/Application-genuine.html' rel='nofollow'>Application.genuine</a> property. If this property returns <span class='doc-keyword'>false</span> then you might notify user that he is using a hacked application or maybe disable access to some functions of your application.
</p>
<p class='vspace'><b>Note:</b> <a class='urllink' href='../ScriptReference/Application-genuineCheckAvailable.html' rel='nofollow'>Application.genuineCheckAvailable</a> should be used along with <span class='doc-keyword'>Application.genuine</span> to verify that application integrity can actually be confirmed. Accessing the <a class='urllink' href='../ScriptReference/Application-genuine.html' rel='nofollow'>Application.genuine</a> property is a fairly expensive operation and so you shouldn't do it during frame updates or other time-critical code. 
</p>
<div class='vspace'></div><h2>Vibration Support</h2>
<p>You can trigger a vibration by calling <a class='urllink' href='../ScriptReference/Handheld.Vibrate.html' rel='nofollow'>Handheld.Vibrate</a>. However, devices lacking vibration hardware will just ignore this call.
</p>
